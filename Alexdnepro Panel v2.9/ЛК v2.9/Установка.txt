Личный кабинет состоит из 2-х частей: клиентской и серверной. Также для работы топа игроков нужно будет заменить некоторые классы в сервисе authd.

	Установка клиент части:
- для работы клиент части нужны установленные пакеты php5-curl php5-gd (php не ниже версии 5.4)
- создаем новую базу данных в MySQL и импортируем туда client_side.sql (кодировка файла в utf-8)
- заливаем на хост содержимое папки client_side
- если файлы заливались через ssh или не от веб пользователя - меняем права (а лучше владельца) файлов/папков так, чтобы у веб пользователя был к ним доступ на запись
- в config_db.php настроить коннект к созданной базе (сохранять в кодировке UTF-8 без BOM)
	$admins - тут айди аккаунтов администраторов
	$admins_ip - тут вставляем свой айпи для первоначальной настройки, под этим айпи любой аккаунт будет с правами админа и без ограничений входа по IP
- пробуем открыть страницу с ЛК (просто открыть не логиниться), если ошибок не будет - переходим к установке сервер части

	Установка сервер части:
- заливаем содержимое папки server_side в папку веб сервера и выставляем на них владельца веб юзера, например chown -R www-data /var/www
- импортируем данные из server_side.sql в базу данных pw
- to_auth_table_xml.txt - данные отсюда добавляем в table.xml (от authd) в конец файла перед </application>
- настраиваем config.php сервер сайда (параметр $act_key берем из клиент панели https://alexdnepro.net/?user_panel)
- проверяем чтобы были установлены php5-gd php5-curl curl (после установки перезапустите апач: service apache2 restart)

	Настройка клиент части:
- если добавлен айпи администратора то при открытии страницы лк на окне логина должна быть шестеренка, нажимаем и настраиваем хотя бы первую вкладку
- пробуем зайти на аккаунт в ЛК
- если не получается - включаем в первой вкладке настроек параметр show_result и смотрим ответы сервера
- настраиваем остальные параметры

	Настройка AUTHD на сервер части:
- в папке protocol_for_auth есть классы (для разных типов auth) которые надо заменить
- делаем бекап своих классов из authd и заменяем новыми
- выполняем инструкции из Системного раздела настроек ЛК (в клиент части) относительно authd
- перезапускаем authd и пробуем зайти в игру
- если при выходе из аккаунта в таблицу top на сервер сайде добавились записи о персонажах - все работает как надо

	Для сервера версии 1.3.6 (сервер сайд) (если версия Вашего сервера выше - данные действия выполнять не нужно!):
- копируем файл getguilds.sh из папки for136 в папку с gamedbd и выдаем права на его запуск
- правим в файле getguilds.sh пути на актуальные (если требуется)
- копируем файл refreshtop.php из папки for136 в папку /var/www
- добавляем в крон данную строку, предварительно подправив в ней пути на актуальные (если требуется)
50 04 * * * /home/gamedbd/getguilds.sh
